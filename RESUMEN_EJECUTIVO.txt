================================================================================
AGRYNC: PLATAFORMA IoT PARA TELEMETRÍA INDUSTRIAL AGROINDUSTRIAL
Resumen Ejecutivo
================================================================================

VISIÓN GENERAL
--------------
Agrync es una plataforma full-stack diseñada para integrar, recolectar y 
gestionar datos de telemetría de dispositivos industriales e IoT en 
instalaciones agroindustriales. El sistema unifica múltiples protocolos 
industriales (Modbus TCP/IP, OPC UA) en una única interfaz web, facilitando 
la monitorización y gestión de sensores sin necesidad de conocimientos 
técnicos especializados.


PROBLEMÁTICA ABORDADA
---------------------
Las plantas agroindustriales enfrentan desafíos críticos:

• Fragmentación de sistemas: cada protocolo requiere herramientas separadas
• Complejidad técnica: configuración manual requiere expertos especializados
• Altos costos: inversión significativa en infraestructura heterogénea
• Falta de escalabilidad: dificultad para añadir nuevos dispositivos
• Ausencia de estándares: incompatibilidad entre sistemas propietarios


SOLUCIÓN PROPUESTA
------------------
Agrync proporciona:

1. INTEGRACIÓN UNIFICADA
   - Soporte nativo para Modbus TCP/IP
   - Servidor OPC UA para interoperabilidad
   - Integración con estándares IoT (FIWARE)
   - Base de datos MongoDB para almacenamiento centralizado

2. INTERFAZ WEB INTUITIVA
   - Gestión visual de dispositivos y sensores
   - Configuración manual o importación masiva (JSON)
   - Dashboard de monitorización en tiempo real
   - Sistema de roles y autenticación

3. DESPLIEGUE SIMPLIFICADO
   - Contenedores Docker para instalación rápida
   - Docker Compose para stack completo
   - Sin dependencias complejas


ARQUITECTURA TÉCNICA
--------------------
Backend:
  • Framework: FastAPI (Python 3.11+)
  • Base de datos: MongoDB con Beanie ODM
  • Protocolos: Modbus TCP/IP, OPC UA
  • Integración: Cliente FIWARE para IoT estándar
  • Autenticación: JWT tokens con RBAC

Frontend:
  • Framework: React 19+ con TypeScript
  • Build tool: Vite (HMR, optimización)
  • API Client: Axios con hooks personalizados
  • UI: Componentes modulares y reutilizables


CARACTERÍSTICAS PRINCIPALES
----------------------------
✓ Lectura automática de registros Modbus (bobinas, entradas, registros)
✓ Servidor OPC UA para exposición de variables a sistemas SCADA
✓ Sincronización bidireccional con plataformas FIWARE
✓ Gestión de múltiples dispositivos y esclavos simultáneos
✓ Configuración en lote mediante archivos JSON
✓ Validación automática de rangos y valores
✓ Sistema de logs y trazabilidad
✓ Control de acceso basado en roles (Admin, Operador, Visualizador)


FLUJO DE OPERACIÓN
------------------
1. Usuario configura dispositivo Modbus (IP, puerto, ID esclavo)
2. Sistema establece conexión TCP/IP automática
3. Lectura periódica de registros configurados
4. Almacenamiento en MongoDB con timestamp
5. Exposición de datos vía OPC UA para clientes externos
6. Reenvío opcional a FIWARE para análisis avanzado
7. Visualización en dashboard web con gráficos en tiempo real


DESPLIEGUE
----------
Opción 1 - Docker Compose (Recomendado):
  $ docker compose up --build
  • Backend: http://localhost:8000
  • Frontend: http://localhost:5173
  • MongoDB: localhost:27017

Opción 2 - Desarrollo Local:
  Backend:
    $ cd agrync_backend
    $ python -m venv .venv && source .venv/bin/activate
    $ pip install -r requirements.txt
    $ uvicorn main:app --reload

  Frontend:
    $ cd agrync_frontend
    $ npm install && npm run dev


REQUISITOS DEL SISTEMA
----------------------
Mínimos:
  • Docker Engine 20.10+ y Docker Compose 1.29+
  • 2GB RAM disponible
  • Puertos 8000, 5173, 27017 libres

Desarrollo:
  • Python 3.11+ (backend)
  • Node.js 18+ (frontend)
  • MongoDB 5.0+


SEGURIDAD
---------
• Autenticación JWT con tokens de corta duración
• Contraseñas hasheadas con algoritmos modernos
• Control de acceso basado en roles (RBAC)
• Certificados opcionales para OPC UA
• Validación de datos con Pydantic (backend) y TypeScript (frontend)
• HTTPS recomendado en producción (vía reverse proxy)


BENEFICIOS CLAVE
----------------
1. REDUCCIÓN DE COSTOS
   - Un solo sistema vs. múltiples herramientas propietarias
   - Infraestructura basada en contenedores (menor mantenimiento)
   - Open source y extensible

2. SIMPLICIDAD
   - Interfaz web intuitiva sin curva de aprendizaje pronunciada
   - Configuración visual vs. programación manual
   - Importación masiva para escalado rápido

3. INTEROPERABILIDAD
   - Estándares abiertos (Modbus, OPC UA, FIWARE)
   - No vendor lock-in
   - Integración con ecosistemas existentes

4. ESCALABILIDAD
   - Arquitectura modular y desacoplada
   - Soporte para múltiples dispositivos simultáneos
   - Preparado para clustering y alta disponibilidad


CASOS DE USO
------------
• Monitorización de invernaderos (temperatura, humedad, CO2)
• Control de sistemas de riego automatizado
• Gestión de silos y almacenes (peso, nivel, temperatura)
• Supervisión de equipos de procesamiento (PLCs industriales)
• Integración con sistemas SCADA existentes
