AGRYNC: IoT PLATFORM FOR INDUSTRIAL AGROINDUSTRIAL TELEMETRY
Executive Summary
================================================================================

OVERVIEW
--------
Agrync is a full-stack platform designed to integrate, collect and manage
telemetry data from industrial and IoT devices in agroindustrial facilities.
The system unifies multiple industrial protocols (Modbus TCP/IP, OPC UA) into
a single web interface, enabling sensor monitoring and management without
requiring specialised technical knowledge.


PROBLEM STATEMENT
-----------------
Agroindustrial plants face critical challenges:

• System fragmentation: each protocol requires separate tools
• Technical complexity: manual configuration requires specialised experts
• High costs: significant investment in heterogeneous infrastructure
• Lack of scalability: difficulty adding new devices
• Absence of standards: incompatibility between proprietary systems


PROPOSED SOLUTION
-----------------
Agrync provides:

1. UNIFIED INTEGRATION
   - Native support for Modbus TCP/IP
   - OPC UA server for interoperability
   - Integration with IoT standards (FIWARE)
   - MongoDB database for centralised storage

2. INTUITIVE WEB INTERFACE
   - Visual management of devices and sensors
   - Manual configuration or bulk import (JSON)
   - Real-time monitoring dashboard
   - Role and authentication system

3. SIMPLIFIED DEPLOYMENT
   - Docker containers for quick installation
   - Docker Compose for full stack
   - No complex dependencies


TECHNICAL ARCHITECTURE
----------------------
Backend:
  • Framework: FastAPI (Python 3.11+)
  • Database: MongoDB with Beanie ODM
  • Protocols: Modbus TCP/IP, OPC UA
  • Integration: FIWARE client for standard IoT
  • Authentication: JWT tokens with RBAC

Frontend:
  • Framework: React 19+ with TypeScript
  • Build tool: Vite (HMR, optimisation)
  • API Client: Axios with custom hooks
  • UI: Modular and reusable components


KEY FEATURES
------------
✓ Automatic reading of Modbus registers (coils, inputs, registers)
✓ OPC UA server for exposing variables to SCADA systems
✓ Bidirectional synchronisation with FIWARE platforms
✓ Management of multiple simultaneous devices and slaves
✓ Batch configuration via JSON files
✓ Automatic range and value validation
✓ Logging and traceability system
✓ Role-based access control (Admin, Operator, Viewer)


OPERATION FLOW
--------------
1. User configures Modbus device (IP, port, slave ID)
2. System establishes automatic TCP/IP connection
3. Periodic reading of configured registers
4. Storage in MongoDB with timestamp
5. Data exposure via OPC UA for external clients
6. Optional forwarding to FIWARE for advanced analysis
7. Visualisation on web dashboard with real-time graphs


DEPLOYMENT
----------
Option 1 - Docker Compose (Recommended):
  $ docker compose up --build
  • Backend: http://localhost:8000
  • Frontend: http://localhost:5173
  • MongoDB: localhost:27017

Option 2 - Local Development:
  Backend:
    $ cd agrync_backend
    $ python -m venv .venv && source .venv/bin/activate
    $ pip install -r requirements.txt
    $ uvicorn main:app --reload

  Frontend:
    $ cd agrync_frontend
    $ npm install && npm run dev


SYSTEM REQUIREMENTS
-------------------
Minimum:
  • Docker Engine 20.10+ and Docker Compose 1.29+
  • 2 GB RAM available
  • Ports 8000, 5173, 27017 free

Development:
  • Python 3.11+ (backend)
  • Node.js 18+ (frontend)
  • MongoDB 5.0+


SECURITY
--------
• JWT authentication with short-lived tokens
• Passwords hashed with modern algorithms
• Role-based access control (RBAC)
• Optional certificates for OPC UA
• Data validation with Pydantic (backend) and TypeScript (frontend)
• HTTPS recommended in production (via reverse proxy)


KEY BENEFITS
------------
1. COST REDUCTION
   - A single system vs. multiple proprietary tools
   - Container-based infrastructure (lower maintenance)
   - Open source and extensible

2. SIMPLICITY
   - Intuitive web interface with a gentle learning curve
   - Visual configuration vs. manual programming
   - Bulk import for rapid scaling

3. INTEROPERABILITY
   - Open standards (Modbus, OPC UA, FIWARE)
   - No vendor lock-in
   - Integration with existing ecosystems

4. SCALABILITY
   - Modular and decoupled architecture
   - Support for multiple simultaneous devices
   - Ready for clustering and high availability
